# Abstract

Large Language Models (LLMs) are increasingly deployed in automated decision-making and software engineering, making it critical to ensure their outputs do not embed or amplify biases. While previous work—such as the paper *"Bias Unveiled: Investigating Social Bias in LLM-Generated Code"*—investigated social bias in LLM-generated code, it limited testing to a constrained list of attributes for particular problems. This study was conducted to determine the broader extent of algorithmic bias in code generated by modern LLMs, specifically investigating whether they encode discriminatory logic across both demographic and non-demographic ("functional") attributes. To answer this, we replicated and extended the original study utilizing both the Google Gemini 2.5 Flash and Grok APIs to generate Python code samples based on 343 comprehensive prompts. We introduced a novel, data-driven framework termed **Smart Combination Testing**. Instead of narrowly testing constrained demographic lists, this framework automatically parses the generated code to identify all utilized input variables and evaluates the function by executing the Cartesian product of every possible attribute value—employing Monte Carlo sampling capped at 100,000 combinations to manage combinatorial explosions—dynamically detecting any output discrepancies. Our expansive auditing framework revealed widespread bias, successfully flagging over 1,400 generated functions as exhibiting discriminatory or inconsistent behavior—a significantly higher detection rate than traditional methods. Crucially, the analysis demonstrated that discriminatory logic is frequently driven by "functional" attributes (such as educational level or college major) rather than demographics alone, alongside explicit instances of religious discrimination. Furthermore, the results showed that the models frequently hallucinated arbitrary decision thresholds—such as hardcoded "magic numbers"—and applied inconsistent logical rules across different samples generated for the exact same prompt. These findings underscore the pervasive and nuanced nature of algorithmic bias embedded within generative AI programming, proving that bias extends far beyond traditional protected classes. The prevalence of functional attribute bias reveals a critical vulnerability when employing LLMs for code generation, as seemingly innocuous variables can trigger discriminatory or unpredictable outcomes in production systems. Ultimately, this research highlights the absolute necessity for robust, scalable auditing frameworks like Smart Combination Testing, and signifies that developers must adopt rigorous, comprehensive evaluation strategies to ensure the fairness and reliability of LLM-integrated software.
